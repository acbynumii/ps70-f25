Minimum Viable Product (MVP)

This week focused on creating a Minimum Viable Product (MVP) for the most challenging aspect of my final project. The project uses a microcontroller program with C++ class structure while avoiding the delay() function. This MVP represents a serious attempt at tackling the core technical challenges of the final project.

My SCARA MVP was all about getting the first DoF (Degree of Freedom) working smoothly, and the hardest part ended up being the mechanical tolerances since I designed the whole robot myself. Microcontroller Program Implementation

Below is the complete microcontroller code that was used in the the MVP demo./

MVP Microcontroller Code
// --- Libraries ---
            #include <Arduino.h>
            
            // --- Pin Assignments ---
            #define STEP_PIN 12   // Step signal
            #define DIR_PIN 14    // Direction signal
            
            // --- Constants ---
            const unsigned long STEP_DELAY_MS = 10;     // Time between steps (ms)
            
            unsigned long lastStepTime = 0;
            
            void setup() {
              Serial.begin(115200);
            
              pinMode(STEP_PIN, OUTPUT);
              pinMode(DIR_PIN, OUTPUT);
            
              digitalWrite(DIR_PIN, HIGH);  // Set direction (change to LOW to reverse)
              digitalWrite(STEP_PIN, LOW);
            
              Serial.println("Starting continuous rotation...");
            }
            
            void loop() {
              unsigned long now = millis();
            
              // Step at the configured cadence (runs continuously)
              if (now - lastStepTime >= STEP_DELAY_MS) {
                digitalWrite(STEP_PIN, HIGH);
                delayMicroseconds(2);       // Step pulse width (adjust if needed)
                digitalWrite(STEP_PIN, LOW);
                lastStepTime = now;
              }
            }

Challenges and Reflection

While building my MVP, the biggest challenges came from the mechanical design and how everything integrated together since I was designing the robot from scratch. Tolerancing was a major issue, and I went through several iterations of 3D printed parts because things either did not fit correctly, were difficult to assemble, or had unexpected friction. I also had trouble properly integrating the rotary bearing, and it took a few design revisions to get it positioned and working smoothly. Through this process, I learned how important it is to think about manufacturing limits early on and to design around critical components from the beginning. Even though my MVP did not reach my original goal, the iterations I made helped me refine the design, make solid progress afterward, and build a much stronger foundation for the final version of the project.
